# TruckMind-Agent — 基于Agentic AI的卡车司机连续找货决策

> 天池 Agent 开发大赛 · 司机找货 Agent 连续决策仿真赛 Baseline

## 🏁 比赛简介

本赛题在**货运业务场景**下构建仿真环境，要求参赛者实现一个**司机找货智能体（Agent）**。系统在每轮交互中提供司机身份信息，智能体须自主获取环境状态与可选行动，在模拟的一个自然月内持续推进决策，在遵守业务约束的前提下实现**月度净收益**最大化，并尽可能满足司机的**个性化偏好**。

### 核心挑战

- **中长期决策**：不是单次接单预测，而是跨时段、多步衔接的连续决策
- **动态环境**：货源、时间、空间不断变化，需要实时感知与响应
- **多目标优化**：兼顾净收益与个性化偏好（如休息时间、夜间不接单等）
- **资源约束**：复赛阶段总仿真运行时长不超过 1 小时，每司机 Token 消耗不超过 500 万

### 三类动作

| 动作 | 含义 | 参数 |
|------|------|------|
| `take_order` | 接单：选定货源并完成承接 | `cargo_id` |
| `reposition` | 空驶：驶向指定区域寻求后续机会 | `latitude`, `longitude` |
| `wait` | 休息：原地等待若干分钟 | `duration_minutes` |

### 评分公式

```
net_income = gross_income - distance_km × cost_per_km - preference_penalty
```

综合得分由**净收益**与**司机个性化偏好满足度**共同构成。

## 🔍 Baseline 代码评价

### 架构设计 ✅

| 方面 | 评价 |
|------|------|
| **模块解耦** | 优秀。`simkit` / `agent` / `server` 三层清晰，通过 `ports.py` 的 Protocol 接口解耦，便于替换决策逻辑 |
| **仿真引擎** | 扎实。`CargoRepository` 使用向量化 Haversine + argpartition 做近邻查询，性能好；pending/online 两级池设计合理 |
| **评测流程** | 完整。从仿真运行→动作日志→收益校验→偏好罚分，链路闭环 |
| **配置管理** | 规范。环境变量优先于配置文件，避免密钥泄露 |

### 决策逻辑 ⚠️（主要短板）

Baseline 的 `ModelDecisionService` 是一个**最简实现**，存在以下明显不足：

1. **无状态决策**：每步决策完全独立，没有跨步记忆或历史信息积累，无法进行中长期规划
2. **信息利用不充分**：
   - 仅传递前 20 条候选货源，未做深度筛选或排序
   - 未传递司机偏好（`preferences`）给模型
   - 未传递累计收益、驾驶时长等关键状态
3. **Prompt 工程粗糙**：
   - System prompt 仅给出输出格式约束，缺乏决策策略引导
   - 未提供业务规则说明（如装货时间窗、空驶成本等）
   - 未提供历史决策上下文
4. **无异常恢复**：模型返回异常时直接抛出错误，无 fallback 策略
5. **无偏好感知**：完全忽略司机个性化偏好，必然导致大量罚分

### 仿真引擎细节 ✅

- 距离计算：Haversine 大圆距离，向量化实现
- 时间推进：分钟级精度，`ceil` 向上取整
- 货源管理：按 `create_time` 排序的 pending 队列 + 按过期时间的 online 堆
- 接单流程：空驶→装货窗等待→干线运输→卸货，完整模拟

## 🎯 下一步优化方向

### 1. 状态管理与记忆系统（高优先级）

```
当前：无状态 → 每步从零开始决策
目标：维护跨步状态 → 支持中长期规划
```

- **历史记忆**：记录已接订单、已行驶路线、收益累计，供后续决策参考
- **日历感知**：追踪当前是第几天、已休息时长、是否满足偏好约束
- **货源记忆**：缓存之前查询过的货源信息，避免重复查询浪费仿真时间

### 2. 偏好感知决策（高优先级）

当前 Baseline 完全忽略司机偏好，对 D006-D010 等有偏好的司机必然产生大量罚分：

- **偏好解析**：从 `preferences` 文本中提取结构化约束（休息时长、禁接时段、回家要求等）
- **约束注入**：将偏好转化为决策时的硬约束或软约束
- **罚分预估**：在决策时预估当前选择的潜在罚分，纳入收益比较

### 3. 多步规划与策略优化（核心竞争力）

```
当前：贪心单步决策 → 只看眼前收益
目标：多步前瞻规划 → 最大化月度总收益
```

- **收益预估模型**：对每个候选订单估算净收益（价格 - 空驶成本 - 干线成本 - 机会成本）
- **区域热度分析**：统计各区域货源密度与价格分布，指导空驶方向
- **时段策略**：根据时段（白天/夜间、工作日/周末）调整决策策略
- **休息调度**：主动规划休息时段，避开低效时段，同时满足偏好约束

### 4. Prompt 工程优化（快速提升）

- **丰富 System Prompt**：加入业务规则、决策策略、偏好约束说明
- **结构化上下文**：提供历史决策摘要、当前日进度、偏好满足状态
- **Few-shot 示例**：提供典型场景下的优秀决策示例
- **思维链引导**：要求模型先分析再决策，提升推理质量

### 5. 货源智能筛选与排序

- **多维度评分**：综合考虑价格、距离、时间窗、偏好匹配度
- **机会成本计算**：接此单意味着放弃的其他机会
- **时效性判断**：装货时间窗是否紧迫，是否需要立即决策

### 6. 工程鲁棒性

- **异常恢复**：模型返回异常时的 fallback 策略（如默认 wait）
- **重试机制**：API 调用失败时的指数退避重试
- **Token 预算管理**：监控累计 Token 消耗，在接近上限时简化决策逻辑
- **超时保护**：单步决策超时时自动选择保守策略

### 7. 高级方向

- **多模型协作**：快速模型做日常决策，强模型做关键决策
- **离线策略学习**：用历史仿真数据训练决策策略网络
- **蒙特卡洛树搜索**：对关键决策点进行多步模拟
- **自适应策略**：根据仿真进度动态调整风险偏好（月初探索 vs 月末保守）

## 📌 关键数据说明

- **仿真时间**：2026-03-01 00:00:00 起，默认 31 天
- **空驶速度**：60 km/h（可配置）
- **货源价格**：原始数据单位为**分**，收益计算时 `/100` 转为**元**
- **距离计算**：Haversine 大圆距离，向上取整到分钟
- **司机数量**：10 位（D001-D010），其中 D006-D010 有个性化偏好
- **偏好罚分**：最高可达数千元/月，对净收益影响巨大
